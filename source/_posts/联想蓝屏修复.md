1. 搜索`注册表编辑器`，路径`HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Power\PowerSettings\54533251-82be-4824-96c1-47b60b740d00\be337238-0d82-4146-a960-4f3749d470c7`, 把Attributes从1改成2(Win11是2，Win10是0)

#### Windows错误码
**找到错误码，直接问AI咋办最快。**

这里简单汇总一些：

##### 类别一：与 Windows 更新、安装和系统组件相关

这类错误通常由系统文件损坏、更新失败或安装问题引起。

| 错误代码 | 常见名称/含义 | 主要原因 |
| :--- | :--- | :--- |
| **0x0** | **操作成功** | 不是错误，表示一个操作（如系统检查点）已成功完成。 |
| **0x14C** | **组件存储损坏** | Windows 更新（LCU）安装失败或中断，导致系统核心文件处于不一致状态。 |
| **0x80070002** | **文件未找到** | 在系统更新或安装过程中，无法找到所需的文件。 |
| **0x80070005** | **拒绝访问** | 进程没有足够的权限访问系统文件或资源。 |
| **0x8007000D** | **数据错误** | 下载的更新文件已损坏或不完整。 |
| **0x800F0922** | **磁盘空间不足/驱动冲突** | 安装更新时 C 盘空间不足，或与显卡等驱动程序冲突。 |
| **0xC1900101** | **安装失败** | 在安装更新或升级系统时，某个驱动程序与新版系统不兼容。 |

<br>

##### 类别二：常见的蓝屏终止代码

这些是直接导致蓝屏死机（BSOD）的代码，通常显示在蓝色屏幕上。

| 错误代码 | 常见名称 | 主要原因 |
| :--- | :--- | :--- |
| **CRITICAL_PROCESS_DIED** | 关键系统进程死亡 | 重要的系统进程被意外终止，可能由恶意软件或系统文件损坏引起。 |
| **SYSTEM_THREAD_EXCEPTION_NOT_HANDLED** | 系统线程异常未处理 | **非常常见**。通常是某个硬件驱动程序存在 Bug 或与系统不兼容。 |
| **IRQL_NOT_LESS_OR_EQUAL** | IRQL 不小于或等于 | 驱动程序试图访问没有权限的内存地址。常见于有问题的驱动程序或内存故障。 |
| **KERNEL_SECURITY_CHECK_FAILURE** | 内核安全检查失败 | 系统内核检测到内部数据损坏。可能与驱动、病毒或内存故障有关。 |
| **PAGE_FAULT_IN_NONPAGED_AREA** | 非分页区域中的页面错误 | 系统无法在指定的内存区域中找到所需数据。通常与有问题的硬件（特别是**内存条**）、驱动或杀毒软件有关。 |
| **VIDEO_TDR_FAILURE** | 视频 TDR 失败 | **非常常见**。显卡驱动程序无响应，导致系统尝试重置显卡时失败。与显卡驱动、过热或硬件故障有关。 |
| **DPC_WATCHDOG_VIOLATION** | DPC 看门狗超时 | 一个驱动程序响应系统请求的时间过长。常见于 SSD 驱动、显卡驱动或电源管理驱动问题。 |
| **NTFS_FILE_SYSTEM** | NTFS 文件系统错误 | 硬盘的文件系统出错。可能是磁盘坏道、连接线问题或突然断电导致。 |
| **BAD_SYSTEM_CONFIG_INFO** | 系统配置信息错误 | 注册表中的系统配置单元（如 `SYSTEM`）已损坏。 |
| **UNEXPECTED_STORE_EXCEPTION** | 意外的存储异常 | 系统在与存储设备（硬盘/SSD）交互时发生严重错误。可能是驱动器本身、其驱动程序或连接线故障。 |

<br>

##### 类别三：与网络和访问相关

| 错误代码 | 常见名称/含义 | 主要原因 |
| :--- | :--- | :--- |
| **0x80070035** | **找不到网络路径** | 计算机无法访问网络位置，可能是 IP 地址错误、防火墙阻止或共享设置问题。 |
| **0x80004005** | **未指定的错误** | 一个泛型错误，可能出现在文件复制、安装软件等多种场景，通常与权限或系统策略有关。 |

<br>

##### 如何利用这个汇总列表？

1.  当看到错误代码时，首先在这个列表中查找。
2.  找到后，查看“主要原因”。这能为您提供一个明确的排查方向。
    *   如果代码属于 **类别一（更新/组件）**，优先考虑使用 `sfc /scannow`、DISM 命令或卸载更新。
    *   如果代码属于 **类别二（蓝屏）**，并且提到了“驱动”，那么您的首要任务就是更新（或回滚）硬件驱动程序，特别是显卡、芯片组和存储驱动。
    *   如果代码指向 **硬件（如内存、硬盘）**，则需要运行相应的诊断工具（如 Windows 内存诊断、硬盘制造商的自检工具）。

### 方法一：卸载最近的质量更新

这是针对`LCU错误`最直接的方法。
1. 在`自动修复`界面，点击 `高级选项`。
2. 进入高级选项后，选择 `疑难解答` > `高级选项` > `卸载更新`。
3. 你会看到两个选项：
    *   卸载最新的质量更新
    *   卸载最新的功能更新
4.  请选择`卸载最新的质量更新`。系统会尝试卸载最近安装的累积更新。这个过程可能需要几分钟，完成后电脑会自动重启。
5.  如果成功，电脑将能够正常启动。之后建议暂停Windows更新，或者选择性地安装更新，避免再次遇到有问题的补丁。

### 方法二：使用命令提示符执行系统文件修复

如果方法一无效，或者`卸载更新`选项不可用，我们可以手动修复系统文件。

1.  同样通过`高级选项`进入 `命令提示符`。
2.  在命令提示符窗口中，依次输入以下命令，并每输完一条就按一次 Enter 键：
    *   `chkdsk C: /f /r`
        *   系统会询问是否计划在下次重启时检查该卷，输入 `Y` 并按回车。
        *   这个命令会检查并修复硬盘错误。
    *   `sfc /scannow /offbootdir=C:\ /offwindir=C:\Windows`
        *   这个命令会在脱机模式下扫描并修复受损的系统文件。这是解决系统文件损坏的关键命令。
    *   `exit`
3.  退出命令提示符，然后选择`继续`或重启电脑，看看问题是否解决。

### 方法三：使用系统还原

如果你之前创建过系统还原点，这是一个非常好的回溯办法。
1.  在`高级选项`中，选择 `系统还原`。
2.  系统会列出可用的还原点。选择一个在问题出现之前的日期创建的还原点。
3.  按照提示完成还原过程。请注意，此操作会撤销在还原点之后安装的程序和系统设置，但你的个人文件（如文档、照片）**通常不会**受影响。

### 方法四：重置此电脑

如果以上方法全部失败，这是核武器选项。它会重装Windows，但你可以选择保留你的文件。
1.  在`疑难解答`界面，选择 `重置此电脑`。
2.  你会看到两个选项：
    *   保留我的文件：删除应用和设置，但保留个人文件。(选这个!)
    *   删除所有内容：清空整个系统盘，相当于恢复出厂设置。(不要选！)
3.  按照提示完成重置过程。这需要较长时间，并且之后你需要重新安装所有应用程序。

## 日常预防措施

本次事后补救，其实应该先设好这些。

### 关闭自动更新
很多问题都是Windows自己不明不白更新导致的，于是打算把自动更新关了。
点击`设置`，搜索`计算机管理`，双击`服务和应用程序`，双击`服务`，往下拉，找到并双击`Windows更新`。
<img src="https://raw.githubusercontent.com/YukinoshitaSherry/qycf_picbed/main/img/20251121033211906.png" style="width:75%">

### 创建系统还原点
按Win键，搜索创建系统还原点(在控制面板)，打开。
点击`配置`,选择`启用系统保护`，同时可调整下还原点对空间的最大使用量，设置完成后点击确定。之后要删除管理还原点都进这里。
点击`创建`，在新弹出的窗口输入一个还原点名称，再点击`创建`。
